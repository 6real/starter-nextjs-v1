# i18n: Internationalization

## Overview

This starter uses a custom dictionary-based i18n system with locale routing via `[lang]` dynamic segment. Translations are stored as JSON files and loaded server-side, with a React context for client-side access.

## Architecture

```
src/
├── i18n/
│   ├── config.ts               # Locales, default locale, validators
│   ├── get-dictionary.ts        # Server-side dictionary loader
│   ├── dictionary-provider.tsx  # Client-side React context
│   ├── use-locale.ts           # Hook to get current locale from URL
│   ├── index.ts                # Barrel export (client-safe)
│   └── dictionaries/
│       ├── fr.json              # French translations
│       └── en.json              # English translations
├── proxy.ts                     # Locale detection + redirect
└── app/
    └── [lang]/                  # Locale-based routing
        ├── layout.tsx           # Loads dictionary, provides context
        └── page.tsx             # Uses translations
```

## Supported Locales

Defined in `src/i18n/config.ts`:

```ts
export const LOCALES = ['fr', 'en'] as const;
export type Locale = (typeof LOCALES)[number]; // 'fr' | 'en'
export const DEFAULT_LOCALE: Locale = 'fr';
```

## Routing

The proxy (`src/proxy.ts`) handles locale detection:

1. Checks if the URL already contains a locale (`/fr/...` or `/en/...`)
2. If not, detects the locale from the `Accept-Language` header
3. Redirects to `/{locale}{path}`

## Dictionary Files

### Structure

```json
{
  "meta": {
    "home": {
      "metaTitle": "My App",
      "metaDescription": "A Next.js starter with i18n"
    }
  },
  "home": {
    "title": "Welcome",
    "description": "A simple Next.js starter with i18n support."
  }
}
```

### Adding Translations

Add matching keys in both `fr.json` and `en.json`:

```json
// en.json
{
  "nav": {
    "home": "Home",
    "about": "About"
  }
}
```

```json
// fr.json
{
  "nav": {
    "home": "Accueil",
    "about": "A propos"
  }
}
```

The `Dictionary` type is automatically inferred from the JSON structure - no manual type definitions needed.

## Usage

### Server Components

Use `getDictionary()` directly:

```tsx
import { getValidLocale } from '@/i18n/config';
import { getDictionary } from '@/i18n/get-dictionary';

const MyPage = async ({ params }: { params: Promise<{ lang: string }> }) => {
  const { lang } = await params;
  const dict = await getDictionary(getValidLocale(lang));

  return <h1>{dict.home.title}</h1>;
};

export default MyPage;
```

### Client Components

Use the `useDictionary()` hook (the dictionary is provided by the layout):

```tsx
'use client';

import { useDictionary } from '@/i18n/dictionary-provider';

const MyClientComponent = () => {
  const { dict } = useDictionary();

  return <p>{dict.home.description}</p>;
};

export default MyClientComponent;
```

### Getting the Current Locale

```tsx
'use client';

import { useLocale } from '@/i18n/use-locale';

const LanguageSwitcher = () => {
  const locale = useLocale(); // 'fr' | 'en'

  return <span>Current: {locale}</span>;
};
```

### Metadata

```tsx
import { getValidLocale } from '@/i18n/config';
import { getDictionary } from '@/i18n/get-dictionary';

import type { Metadata } from 'next';

export async function generateMetadata({
  params,
}: {
  params: Promise<{ lang: string }>;
}): Promise<Metadata> {
  const { lang } = await params;
  const dict = await getDictionary(getValidLocale(lang));
  return {
    title: dict.meta.home.metaTitle,
    description: dict.meta.home.metaDescription,
  };
}
```

## Adding a New Locale

1. Add the locale to `LOCALES` in `src/i18n/config.ts`:

```ts
export const LOCALES = ['fr', 'en', 'es'] as const;
```

2. Add the locale-to-lang mapping:

```ts
export const LOCALE_TO_LANG: Record<Locale, string> = {
  fr: 'fr-FR',
  en: 'en-US',
  es: 'es-ES',
};
```

3. Add the dictionary loader in `src/i18n/get-dictionary.ts`:

```ts
const dictionaries = {
  fr: () => import('./dictionaries/fr.json').then(m => m.default),
  en: () => import('./dictionaries/en.json').then(m => m.default),
  es: () => import('./dictionaries/es.json').then(m => m.default),
} as const;
```

4. Create `src/i18n/dictionaries/es.json` with the same structure as `fr.json`.

## Key Rules

- **Never create manual Dictionary types** - the type is inferred from the JSON
- **All user-facing strings must be translated** via `dict`
- **Import `Dictionary` type** from `@/i18n/get-dictionary` when needed
- **`get-dictionary.ts` is server-only** - do not import it in client components
